Wichtige Sicherheitsmaßnahmen

  • GitHub 2FA aktivieren

  • wenige Browser Extensions nutzen

Bereits korrekt umgesetzt

  • anonymisierte GitHub-Identität
  • cleanes neues Repository
  • Netlify statt eigener Server
  • HTTPS Hosting
  • kleine Angriffsfläche

Gesamtbewertung

Die aktuelle Infrastruktur ist für kleine öffentliche Browsergames:

  • modern
  • relativ sicher
  • einfach wartbar
  • privacy-orientiert
  • gut reproduzierbar

Die größte reale Gefahr aktuell sind versehentliche Datenleaks — nicht externe Hacker.
